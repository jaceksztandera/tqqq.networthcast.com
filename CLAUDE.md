## What this project is

A client-side-only backtesting simulator for leveraged ETF strategies. All logic runs in the browser. No server, no build step — just static files served from the repo root.

---

## File layout

```
index.html          — single-page app shell; all <script> and <link> tags live here
styles.css          — all CSS
js/
  data.js           — TSV loaders, buildDaily(), column layout
  utils.js          — shared formatting helpers (fmt, fmtFull, sliderToMonthly, …)
  simulate.js       — all strategy simulation functions
  metrics.js        — computeMoneyWeightedReturn, computeDailyMaxDrawdown
  chart.js          — Chart.js wiring, render(), panel HTML builders
  controls.js       — SLIDER_IDS, APP_VERSION, event listeners
  analytics.js      — STRATEGY_REGISTRY, STRATEGY_KEY_TO_DATASET_IDX, computeMaxDrawdown
  init.js           — Promise.all data loads, calls render() on first paint
  saved-configs.js  — saved/custom strategy persistence
  preview-dropdown.js — preview chip dropdown logic
tests/
  sim.cjs           — Node unit tests for simulate.js + metrics.js
  suite.cjs         — Playwright end-to-end UI tests (requires a running static server)
data/               — TSV price and distribution files (generated by update_data.py)
                      Includes rate files: fed-funds-effective.tsv, short-rates.tsv, t-bill-3mo.tsv
update_data.py      — data refresh/rebuild script (requires .venv)
```

## APP_VERSION and cache busting

**Both must be kept in sync** every time any `js/*.js` or `styles.css` changes:

1. `APP_VERSION` in `js/controls.js` — increment by 1.
2. All `?v=N` query strings on `<script>` and `<link>` tags in `index.html` — change to the same number.

Current version: **44**. Mismatch causes a "new version — reset saved data" banner for returning users.

When adding a new slider/select control, also add its element ID to `SLIDER_IDS` in `controls.js` and wire an event listener at the bottom of that file.

## Running tests

```bash
node tests/sim.cjs
```

All 91 tests must pass. The suite tests `simulate.js` and `metrics.js` in Node (no browser). Run after any change to those two files.

Syntax-check all JS before testing:

```bash
node --check js/simulate.js js/chart.js js/controls.js js/analytics.js
```

For end-to-end UI tests (Playwright, requires Chromium):

```bash
python3 -m http.server 8753 &   # serve from repo root
node tests/suite.cjs             # BASE=http://host:port to override
```

## Regenerating data

```bash
# One-time venv setup
python3 -m venv .venv && .venv/bin/pip install yfinance --quiet

# Full rebuild (re-fetches all real data, re-synthesizes everything)
.venv/bin/python update_data.py --rebuild

# Incremental refresh (appends only new bars)
.venv/bin/python update_data.py
```

`--rebuild` re-synthesizes all TSVs under `data/`, calibrates JEPQ CC drag from real price data, and writes `data/jepq-distributions.tsv`.

## Key invariants

- **`computeDailyMaxDrawdown` returns an object** `{ pct, peakDate, troughDate }` where `pct` is a fraction (0–1). The consumer at `window._strategyMetrics` does `dd.pct * 100`. Do **not** multiply the return value by 100 before storing it.
- **`simulateTQQQJEPQ` log fields**: growth leg is `ulShares` / `ulVal` / `ulPrice`; income leg is `jepqShares` / `jepqVal` / `jepqPrice`.
- **Envelope ghost lines start at dataset index 15**, not 14. The `while` loop guard in `render()` is `< 15 + envelopeShiftCount`.
- **`jepqDistributionMap`** is a global `Map<YYYY-MM-DD, amountPerShare>` populated in `init.js`.

## Constraints

- **`git commit` is blocked** — the user has a Bash deny rule for `git commit:*`. Never attempt to commit; instruct the user to commit manually.
- **No build step** — do not introduce a bundler, transpiler, or npm-based build. All JS is loaded directly by the browser.
- **No server-side code** — the site is fully static; `update_data.py` only touches local files.
